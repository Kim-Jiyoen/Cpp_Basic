# 클래스

- 객체 지향 프로그램의 대표적인 기법
- 구조체로부터 업그레이드된 기법
  - 구조체 : 변수만으로 구성된 집합체
  - 클래스 : 변수와 함수로 구성된 집합체

- 객체지향 프로그래밍의 특징
  - 추상화 
  - 캡슐화 
  - 정보 은닉 
  - 상속성
  - 다형성
  
- 클래스의 구성
 - 변수: 멤버 변수 / 프로퍼티(ex.차 구성요소 - 차 컬러, 창문, 바퀴 등)
 - 함수: 멤버 함수 / 메소드(ex.차 기능 - 라디오, 에어컨 작동 등)
 
- 클래스의 의미
  - 클래스
    - 설계도 => 실체 없음. ****
  - 객체 / 인스턴스 
    - 클래스를 통해서 생성된 실체
     
구조체도 생성하면 구조체 타입으로 변수를 선언해야 것처럼
클래스도 설계도를 바탕으로 객체 인스턴스(실체)를 만들어야함

## 생성자(Constructor)와 소멸자(Destructer)

### 생성자

 - 클래스 타입의 변수가 생성될 때 초기화하는 함수
 - 초기화 기능 / 초기화가 진행된 후 연계해서 다른 기능 동작
 
 ### 소멸자
 - 클래스 쓰임새가 다했거나 프로그램 종료 시 생성되었던 클래스를 해제하는 함수
 - 소멸 기능 / 클래스 소멸이 진행된 후 연계해서 다른 기능 동작
